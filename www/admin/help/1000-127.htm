Anrufarchive können eine beliebige Anzahl von Anrufinformationen archivieren, die z.B. vom Anrufmonitor***a-1-3 bereitgestellt werden. Wichtig: Voraussetzung für die Nutzung von Telefonie-Funktionen ist die Konfiguration eines Telefonie-Gateways***a-1-2 und ggf. des Anrufmonitors***a-1-3. Hinweis: Das Anlegen von mehr als einem Anrufarchiv ist für die meisten Anwendungsfälle nicht erforderlich bzw. nicht sinnvoll.

Eigenschaften

  • Name: frei wählbarer Name des Anrufarchivs
  • KO: Status
    • bei jeder Änderung des Datenbestandes an diesem Archiv wird dieses KO auf die Gesamtanzahl der Archiveinträge gesetzt:
    • beim Start von EDOMI
    • beim Hinzufügen/Entfernen eines Eintrags
    • beim automatischen Löschen von Einträgen (s.u.)
  • Speicherdauer: legt fest, ob und wann Einträge automatisch gelöscht werden
    • Archiveinträge werden grundstäzlich mit einem Zeitstempel archiviert
    • ist ein Archiveintrag älter als die festgelegte Speicherdauer, wird der Archiveintrag beim Start von EDOMI und täglich um 00:00:00 Uhr aus dem Archiv gelöscht

Hinzufügen und Entfernen von Archiveinträgen

Archiveinträge werden ggf. automatisch vom Anrufmonitor***a-1-2 und über entsprechende Befehle***1007 hinzugefügt bzw. entfernt. Zudem können Logikbausteine***r-0 weitere Funktionen zur Verwaltung von Archiveinträgen bereitstellen. Hinweis: Archiveinträge werden u.U. automatisch gelöscht, sofern die Eigenschaft "Speicherdauer" (s.o.) entsprechend konfiguriert wurde.

Struktur von Archiveinträgen

Archiveinträge werden jeweils mit einem eindeutigen Zeitstempel versehen (Datum, Uhrzeit und Mikrosekunden). Als Anrufinformationen werden die Rufnummer, die Anrufrichtung (eingehend/ausgehend) und einige Metadaten archiviert.
