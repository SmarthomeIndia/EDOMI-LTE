Entwicklung von Visuelementen***r-3-0VSE-Dateien/-ordner

Ein Visuelement (VSE) wird durch genau eine Datei definiert (z.B.: 1000_vse.php). Der Dateiname muss stets diesem Schema folgen:
  • <ID>_vse.php
    • <ID>: eine eindeutige(!) ID im Bereich 1000..99999999 (für "Eigene Visuelemente")
  • die Datei muss im Verzeichnis /www/admin/vse gespeichert werden
Ein VSE kann entweder mit einem beliebigen Editor direkt als Datei bearbeitet werden, oder mit Hilfe der Konfiguration***1000-160. Letzteres ist empfehlenswert, da beim Speichern eines VSE unmittelbar diverse Prüfungen vorgenommen werden. Wichtig:
  • Wurde ein VSE mit einem externen Editor erstellt bzw. bearbeitet, müssen anschließend alle VSE-Definitionen neu eingelesen werden (Verwaltung/Visuelemente/Visuelemente neu einlesen***103-16).
  • Wie bei allen anderen projektbezogenen Einstellungen auch, werden Änderungen an einem VSE erst nach einer Projektaktivierung***103-13 bzw. Visuaktivierung***103-15 im Live-Projekt berücksichtigt. Die Aktivierung erzeugt eine unabhängige VSE-Instanz.
  • Visuelemente werden nicht(!) projektbezogen verwaltet, sondern sind ein Bestandteil der EDOMI-Installation. Auch Eigene Visuelemente***r-3-0 werden stets in diesem globalen Kontext angelegt und verwaltet.

Ordner-IDs

Zur Organisation der VSE sind die folgenden Ordner-IDs fest vorgegeben. Es können grundsätzlich keine Visuelemente oder neue Ordner hinzugefügt werden, eigene Visuelemente werden stets im Ordner "Eigene Visuelemente" abgelegt.
Ordner-ID Bezeichnung Hinweis
160 Visuelemente nur für System-LBS, d.h. hier können keine eigenen VSE abgelegt werden

163 Archive
164 Sonstige
