Logikbausteine (allgemein)

Sämtliche Logikfunktionen basieren auf Logikbausteinen als kleinste funktionale Einheit einer Logik***r-1. Einzelne Logikbausteine können im Logikeditor***1 miteinander verbunden werden und somit eine komplexe Gesamtfunktionalität bereitstellen. Die Darstellung (Farbschema) der Logikbausteine im Logikeditor***1 kann in der Basis-Konfiguration***a-1 angepasst werden. Die Eingangsbezeichnungen (z.B. "E1") von Logikbausteinen werden im Logikeditor***1 u.U. nach einem bestimmten Schema farblich hervorgehoben. Dieses Schema ist jedoch nicht zwingend und hat prinzipiell keinen Einfluss auf die Funktionalität eines Logikbausteins. EDOMI-Logikbausteine folgen aber grundsätzlich dem folgenden Farbschema (die Darstellung Eigener Logikbausteine***r-0-0 ist abhängig von deren Implementierung):
  • grün: der Eingang triggert i.d.R. den Logikbaustein (z.B. startet ein bestimmter Eingangswert einen Timer-Baustein)
  • gelb: der Eingang repräsentiert eine besondere oder wichtige Option (z.B. definiert der Eingangswert einen wichtigen Parameter)
  • orange: der Eingang ist besonders wichtig bzw. beachtenswert (z.B. definiert der Eingangswert einen zwingend erforderlichen Parameter)
  • grau: keine besondere Hervorhebung
Hinweis: Dieses Schema ist nicht zwingend, d.h. einige Logikbausteine weichen u.U. hiervon ab. Auch ist ein Umkehrschluss nicht generell zulässig, d.h. auch ein "nicht-grüner" Eingang kann u.U. den Logikbaustein triggern. Wichtig: Loginbausteine werden nicht(!) projektbezogen verwaltet, sondern sind ein Bestandteil der EDOMI-Installation. Auch Eigene Logikbausteine***r-0-0 werden stets in diesem globalen Kontext angelegt und verwaltet, jedoch können diese Logikbausteine bei Bedarf projektbezogen in Ordnern strukturiert werden.
