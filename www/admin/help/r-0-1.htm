Logikbausteine (allgemein)***r-0Entwicklung von Logikbausteinen***r-0-0LBS-Dateien/-ordner

Ein LBS wird durch genau eine Datei definiert (z.B.: 14000020_lbs.php). Der Dateiname muss stets diesem Schema folgen:
  • xxyyyyyy_lbs.php
    • xx: die interne Ordner-ID (s.u.)
    • yyyyyy: eine eindeutige(!) 6-stellige ID, ggf. mit führenden Nullen (z.B. 000001)
  • der Dateiname hat demnach stets eine Länge von 8 Ziffern zzgl. "_lbs.php"
  • die Datei muss im Verzeichnis /www/admin/lbs gespeichert werden
Ein LBS kann entweder mit einem beliebigen Editor direkt als Datei bearbeitet werden, oder mit Hilfe der Konfiguration***1000-12. Letzteres ist empfehlenswert, da beim Speichern eines LBS unmittelbar diverse Prüfungen vorgenommen werden. Wichtig:
  • Wurde ein LBS mit einem externen Editor erstellt bzw. bearbeitet, müssen anschließend alle LBS-Definitionen neu eingelesen werden (Verwaltung/Logikbausteine/Logikbausteine neu einlesen***103-8).
  • Wie bei allen anderen projektbezogenen Einstellungen auch, werden Änderungen an einem LBS erst nach einer Projektaktivierung***103-13 im Live-Projekt berücksichtigt. Die Projektaktivierung erzeugt eine unabhängige LBS-Instanz.
  • Loginbausteine werden nicht(!) projektbezogen verwaltet, sondern sind ein Bestandteil der EDOMI-Installation. Auch Eigene Logikbausteine***r-0-0 werden stets in diesem globalen Kontext angelegt und verwaltet, jedoch können diese Logikbausteine bei Bedarf projektbezogen in Ordnern strukturiert werden.
