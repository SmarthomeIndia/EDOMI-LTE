Administrationsseite***0-1Visueditor***2Visuelemente***1002Design

Mit Hilfe von Designs wird das Erscheinungsbild von Visuelementen***1002 in der Visualisierung***b-0 definiert. Grundsätzlich werden zwei Arten von Designs unterschieden:
  • Design (bzw. "Basis-Design"): diese Design-Eigenschaften werden standardmäßig für das Visuelement verwendet
    • existiert kein zum KO-Wert passendes dynamisches Design (s.u.), wird dieses "Basis-Design" angewendet
    • Hinweis: Für jedes Visuelement kann genau 1 "Basis-Design" definiert werden.
  • dynamische Designs: diese Design-Eigenschaften werden nur dann für das Visuelement verwendet, wenn der aktuelle Wert von KO1 oder ggf. KO3 des Visuelments innerhalb eines definierten Bereichs liegt
    • liegt der aktuelle Wert von KO1 oder ggf. KO3 innerhalb des angegebenen Bereichs, wird das entsprechende dynamische Design angewendet
    • existiert kein zum KO-Wert passendes dynamisches Design, wird das "Basis-Design" angewendet (s.o.)
    • Hinweis: Für jedes Visuelement kann eine beliebige Anzahl an dynamischen Designs verwaltet werden.
    • grundsätzlich wird der Wert von KO1 zur Steuerung des dynamischen Designs verwendet, jedoch wird bei konfiguriertem KO3 die Steuerung von KO3 übernommen:
      • KO3 wird ausschließlich zur Steuerung eines dynamischen Designs***1003 verwendet
      • wenn KO3 angegeben wurde, wird ein dynamisches Design durch dieses KO3 gesteuert
      • wenn KO3 nicht angegeben wurde, wird ein dynamisches Design durch das KO1 gesteuert
    • Wichtig:
      • ein dynamisches Design ersetzt stets sämtliche(!) Eigenschaften des "Basis-Designs" (die Eigenschaften werden also nicht(!) zusätzlich verwendet)
      • Ausnahme: Wird die Eigenschaft "Beschriftung" nicht angegeben, wird die entsprechende Angabe aus den Visuelement-Einstellungen***1002 verwendet.

Designvorlage

Ermöglicht die optionale Auswahl einer Designvorlage***1000-24, die für dieses Design verwendet werden soll. Die Auswahl einer Designvorlage führt dazu, dass alle Design-Eigenschaften der Vorlage auf das Visuelement angewendet werden. Zusätzlich können jedoch auch individuelle Eigenschaften angegeben werden (s.u.) - diese Angaben werden dann anstelle der entsprechenden Eigenschaften aus der Vorlage angewendet. Mit der Schaltfläche "Eingaben zurücksetzen" werden alle individuellen Eingaben zu den jeweiligen Eigenschaften zurückgesetzt. Dies kann z.B. hilfreich sein, wenn zunächst ein individuelles Design definiert wurde, zu einem späteren Zeitpunkt jedoch ausschließlich eine Vorlage verwendet werden soll. Hinweis: Diese Option steht beim Bearbeiten einer Designvorlage***1000-24 nicht zu Verfügung.

Dynamisches Design

Diese Eigenschaften stehen nur zu Verfügung, wenn eine Designvorlage***1000-24 oder ein dynamisches Design eines Visuelement bearbeitet bzw. erstellt wird. Hinweis: Beim Erstellen bzw. Bearbeiten einer Designvorlage***1000-24 wird durch die nachfolgenden Angaben festgelegt, ob die Designvorlage ein "Basis-Design" oder ein dynamisches Design repräsentiert: Werden keine Angaben in den Feldern "von KO-Wert / bis KO-Wert" gemacht, wird die Designvorlage als "Basis-Design", andernfalls als dynamisches Design interpretiert.
  • von KO-Wert / bis KO-Wert: diese Angaben definieren einen Wertebereich für das KO1 bzw. KO3 des Visuelements***1002
    • liegt der aktuelle Wert von KO1 bzw. KO3 innerhalb dieses Wertebereichs, wird dieses dynamische Design verwendet
    • es müssen stets beide Werte angegeben werden ("von KO-Wert" und "bis KO-Wert"), um einen gültigen Wertebereich zu definieren
    • Überschneidungen von Wertebereichen unterschiedlicher dynamischer Designs führen u.U. zu einer "zufälligen" Auswahl des dynamischen Designs und sind daher zu vermeiden
    • Hinweis: die Option "Wertebereich-Datentyp" des Visuelements***1002 legt den Datentyp des angegebenen Wertebereichs fest
      • automatisch: je nach Datentyp von KO1 bzw. KO3 wird ein nummerischer Vergleich oder ein Stringvergleich des KO-Wertes mit dem angegebenen Wertebereich durchgeführt
        • Integer/Float-Datentypen werden nummerisch verglichen, z.B. ist "321" größer als "123"
        • String-Datentypen (z.B. Datum, Uhrzeit, Hexadezimale Werte, etc.) werden als String verglichen, z.B. ist "EDOMI" größer als "ABCDE"
        • beim Datentyp "Variant" wird in Abhängigkeit des aktuellen KO-Wertes entweder ein nummerischer oder ein Stringvergleich durchgeführt
      • Zahl: der Wertebereich wird als Zahl interpretiert, es erfolgt stets ein nummerischer Vergleich mit dem KO-Wert
      • String: der Wertebereich wird als String (Text) interpretiert, es erfolgt stets ein String-Vergleich mit dem KO-Wert
  • Beschriftung: ersetzt ggf. die Eigenschaft "Beschriftung" des Visuelements***1002
    • wird dieses Feld leer belassen, bleibt die Eigenschaft "Beschriftung" des Visuelements unverändert bestehen
    • in dieser Eigenschaft können Variablen, Funktionen und Formeln***r-3 verwendet werden
    • Hinweis: Je nach Visuelement hat dieses Feld eine unterschiedliche Bedeutung***1002.
    • Hinweis: Die maximale Gesamtlänge beträgt 10.000 Zeichen.
  • Schaltfläche "Basis-Design übernehmen":
    • sämtliche Eigenschaften werden aus dem "Basis-Design" (s.o.) des Visuelements übernommen
    • Wichtig: Eine Eigenschaft wird nur übernommen, wenn das entsprechende Ziel-Feld leer ist.
    • Hinweis: Diese Schaltfläche steht beim Bearbeiten einer Designvorlage***1000-24 nicht zu Verfügung.

Eigenschaften

Die nachfolgenden Eigenschaften legen das Erscheinungsbild des gesamten Visuelements, bzw. einzelner Bereiche des Visuelements fest. Wichtig: Sämtliche Angaben ersetzen ggf. solche aus einer zugewiesenen Designvorlage***1000-24 (s.o.): Ist z.B. in der zugewiesenen Designvorlage die Eigenschaft "Drehung" auf den Wert 45 (Grad) gesetzt und in den Eigenschaften auf 10 (Grad), wird das Visuelement um 10 Grad gedreht. Wird diese Angabe in den Eigenschaften jedoch leer belassen, wird das Visuelement um 45 Grad gedreht. Hinweise: In allen hellgrün hinterlegten Eingabefeldern können Variablen, Funktionen und Formeln***r-3 verwendet werden (bzw. bei Farbauswahl-Feldern können dynamische Farben ausgewählt werden). Die maximale Gesamtlänge für jede der nachfolgenden Eigenschaften beträgt 500 Zeichen.
  • Allgemeine Eigenschaften:
    • X-Position/Y-Position: relative(!) Positionierung (in Pixeln)
      • das Visuelement wird relativ zu seiner Position um diesen Betrag in X- bzw. Y-Richtung verschoben
    • Breite/Höhe: relative(!) Skalierung (in Pixeln)
      • die Breite bzw. Höhe des Visuelements wird um diesen Betrag erweitert bzw. verringert
    • Drehung: Rotation (Winkel in Grad)
      • das Visuelement wird um diesen Winkel rotiert
      • die Rotation erfolgt im Uhrzeigersinn, d.h. ein positivert Winkel dreht das Visuelement "rechts herum"
      • das Rotationszentrum ist stets das geometrische Zentrum des Visuelements
    • Opazität: Opazität (0..1)
      • bestimmt die Sichtbarkeit des Visuelements auf einer Skala von 0 bis 1 (z.B. "0"=unsichtbar, "0.5"=halbtransparent, "1"=vollständig sichtbar)
      • Hinweis: Wird keine Opazität angegeben, wird das Visuelement vollständig sichtbar angezeigt (keinerlei Transparenz).
      • Wichtig: Eine Opazität von "0" blendet das Visuelement vollständig aus, d.h. das Visuelement wird dann auch nicht auf Nutzerinteraktionen (z.B. Klicks) reagieren!
  • Farbe und Hintergrund: legt Vorder- und Hintergrundfarbe (und ggf. ein Hintergrundbild) des Visuelements fest
    • Farbe: Auswahl einer Vordergrundfarbe***1000-26 für die Beschriftung und Gestaltung des gesamten Visuelements
      • je nach Visuelement werden die Beschriftung und/oder weitere Elemente in der ausgewählten Farbe dargestellt (z.B. Schieberegler***1002-13, Diagramme***1002-21, uvm.)
      • Hinweis: Wird keine Vordergrundfarbe angegeben, wird die Standard-Vordergrundfarbe (schwarz) verwendet.
    • Hintergrundfarbe: Auswahl einer Hintergrundfarbe***1000-25 für das gesamte Visuelement
      • wird keine Hintergrundfarbe angegeben, wird der Hintergrund transparent dargestellt
      • die Kombination von Hintergrundfarbe und -bild ist möglich (z.B. werden transparente Bereiche eines PNG-Bildes dann in der Hintergrundfarbe dargestellt)
    • Hintergrundbild: Auswahl eines Bildes***1000-28, das als Hintergrundbild für das gesamte Visuelement verwendet wird
      • die Kombination von Hintergrundfarbe und -bild ist möglich (z.B. werden transparente Bereiche eines PNG-Bildes dann in der Hintergrundfarbe dargestellt)
      • Hinweis: Das Hintergrundbild wird i.d.R. an die Größe des Visuelements angepasst. Dies erlaubt z.B. die Verwendung von Bildern, die in einer deutlich höheren Auflösung vorliegen ("Retina"-Bildschirme). Einige Visuelemente erlauben eine individuelle Skalierung des Hintergrundbildes (z.B. das Universalelement***1002-1).
  • Text: legt das Erscheinungsbild sämtlicher Texte (z.B. die Beschriftung) des Visuelements fest
    • Schriftart: Auswahl einer Schriftart***1000-150 für die Beschriftung und Gestaltung des gesamten Visuelements
      • Hinweis: Wird keine Schriftart angegeben, wird die Standard-Schriftart aus der Basis-Konfiguration***a-1 verwendet.
    • Schriftgröße: definiert die Schriftgröße (in Pixeln) für die Beschriftung und Gestaltung des gesamten Visuelements
      • Hinweis: Wird keine Schriftgröße angegeben, wird die Standard-Schriftgröße aus der Basis-Konfiguration***a-1 verwendet.
    • Stil / Stärke: legt diverse Schriftattribute für die Beschriftung des Visuelements fest
    • Innenabstand: definiert den Innenabstand (in Pixeln) für die Beschriftung des Visuelements
      • Hinweis: Einige Visuelemente ignorieren diese Eigenschaft.
    • Ausrichtung: definiert die horizontale Textausrichtung der Beschriftung des Visuelements
      • Hinweis: Die vertikale Ausrichtung der Beschriftung ist stets zentriert.
      • Hinweis: Einige Visuelemente ignorieren diese Eigenschaft.
  • Textschatten: sämtliche Texte (z.B. die Beschriftung) des Visuelements können ggf. mit einem Schatten hinterlegt werden
    • X-/Y-Abstand: legt die Distanz des Schattens zur Beschriftung fest (in Pixeln)
    • Unschärfe: gibt den Grad der Unschärfe des Schattens an (in Pixeln)
    • Farbe: der Textschatten wird mit dieser Vordergrundfarbe***1000-26 erzeugt
    • Hinweis: Es müssen entweder alle oder keine der o.g. Eigenschaften angegeben werden.
  • Schatten: das gesamte Visuelement kann ggf. mit einem Schatten hinterlegt werden
    • X-/Y-Abstand: legt die Distanz des Schattens zum Visuelement fest (in Pixeln)
    • Unschärfe: gibt den Grad der Unschärfe des Schattens an (in Pixeln)
    • Überstand: definiert die Breite eines Bereichs (in Pixeln), der vollständig in der Schattenfarbe angezeigt wird - ähnlich eines Rahmens
    • Farbe: der Schatten wird in dieser Vordergrundfarbe***1000-26 dargestellt
    • Hinweis: Es müssen entweder alle oder keine der o.g. Eigenschaften angegeben werden.
  • Rahmen: ermöglicht das Einfärben jeder der 4 Kanten des Visuelements in der angegeben Farbe
    • jeder Kante kann eine individuelle Vordergrundfarbe***1000-26 zugewiesen werden
    • wird einer Kante keine Farbe zugewiesen, wird diese Kante von der Umrahmung ausgeschlossen
    • Breite: Rahmenstärke für alle Kanten, denen eine Farbe zugewiesen wurde (in Pixeln)
    • Muster: legt das Erscheinungsbild des Rahmens für alle Kanten gemeinsam fest
  • Animation: ermöglicht das Animieren des gesamtes Visuelements mit Hilfe einer konfigurierten Animation***1000-27
    • Animation: Auswahl der gewünschten Animation***1000-27
    • Dauer: bestimmt die Dauer der Animation in Sekunden
    • Anzahl: legt fest, wie oft die Animation abgespielt werden soll (die Angabe von "0" führt dazu, dass die Animation unendlich wiederholt wird)
    • Hinweis: Es müssen entweder alle oder keine der o.g. Eigenschaften angegeben werden.
  • Eckenradius: ermöglicht das Abrunden jeder der 4 Ecken des Visuelements mit dem angegeben Radius (in Pixeln)
    • Hinweis: Hat ein Visuelement z.B. die Größe 100x100 Pixel, führt ein Eckenradius von jeweils 50 Pixeln (oder mehr) zu einem kreisrunden Visuelement.
  • Zusatzeigenschaften: einige Visuelemente können zusätzliche Farben oder Bilder nutzen (z.B. Universalelement, Zeitschaltuhr, etc.)
    • Zusatzvordergrundfarbe 1/2: Auswahl einer zusätzlichen Vordergrundfarbe***1000-26
    • Zusatzhintergrundfarbe 1/2: Auswahl einer zusätzlichen Hintergrundfarbe***1000-25
    • Zusatzbild 1/2: Auswahl eines zusätzlichen Bildes***1000-28
    • Hinweis: Die Zusatzeigenschaften werden nur von einigen wenigen Visuelementen genutzt und werden ansonsten ignoriert.
  • Eigene CSS-Eigenschaften: ermöglicht die Angabe von beliebigen zusätzlichen CSS-Eigenschaften
    • die hier angegebenen CSS-Eigenschaften müssen jeweils mit einem Semikolon abgeschlossen werden
    • Beispiel: Visuelement perspektivisch (3D) darstellen
      • -webkit-transform: perspective(1000px) rotateX(15deg) rotateY(30deg) rotateZ(15deg);
    • Wichtig: Die Angaben werden nicht(!) auf eine korrekte Syntax etc. geprüft!
    • Hinweis: Die maximale Gesamtlänge beträgt 1000 Zeichen.
